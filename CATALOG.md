## patterns

- **`canary-promotion`** — Decide whether creation version v_candidate replaces v_current: run both against the same eval set, compare win/regression rates, and promote through a gate — or reject with findings. The self-evolution loop, made a reviewable procedure.
- **`context-handoff`** — When a working agent approaches its context ceiling, it finishes the current atomic step, writes a complete handoff packet, and yields to a fresh successor that continues from the packet alone — before quality degrades, not after.
- **`evaluator-optimizer`** — A generator produces an artifact; an evaluator audits it with objective checks; failures feed back to the generator until pass or budget exhausted.
- **`hitl-checkpoint`** — Run until a declared gate before a consequential action; freeze all state at the gate; resume, adjust, or abort based on a human verdict delivered through the runtime's approval stance.
- **`long-horizon-run`** — Pursue an objective across many work episodes without supervision: each episode does bounded work, checkpoints its state, and emits a heartbeat; interruption at any point resumes from the last checkpoint; stalls and budget exhaustion escalate instead of spinning.
- **`multi-agent-debate`** — Several agents argue distinct assigned perspectives over a decision; after bounded rounds of critique, a moderator synthesizes one recommendation with the disagreements preserved.
- **`orchestrator-workers`** — An orchestrator invents the subtask list at runtime from the objective, dispatches each subtask to a fitting worker profile, integrates results, and may re-plan once from what came back.
- **`parallelization`** — Split a divisible task into independent shards, run the same worker over each concurrently, reduce the shard results into one consolidated output.
- **`planner-executor`** — A planner turns the objective into an explicit, reviewable plan artifact (typed steps with preconditions and effect classes); the plan is approved as a whole; an executor runs it step-by-step, re-validating preconditions and returning to the planner on deviation.
- **`prompt-chaining`** — Fixed linear stages: each stage's typed output is the next stage's input; a gate check between stages stops a bad intermediate from propagating.
- **`routing`** — Single entry point: classify the request's intent and hand it to exactly one specialist; consolidate and return its result.
- **`saga-compensation`** — Execute a sequence of external-effect steps where each step's compensation is registered BEFORE the step runs; on failure at step k, execute compensations k-1..1 in reverse order and report the unwind.
- **`tracer-bullet-decomposition`** — Break an accepted spec or plan into tracer-bullet tickets — vertical slices that each cut a narrow but complete path through every layer, demoable on their own, sized to one fresh context window, each declaring its blocking edges — then publish them so sessions can work the frontier.
- **`triage`** — Move incoming proposals (issues, requests, agent-generated suggestions) through a small state machine — gather, recommend, verify, grill if needed, apply — until each carries exactly one category and one state; every rejection writes its reason to a rejection log so it is never re-proposed.
- **`wayfinder`** — Plan an effort too big for one agent session as a shared map of typed decision tickets on a tracker; each session either charts the map or resolves exactly one ticket, graduating fog into new tickets, until the way to the destination is clear.
